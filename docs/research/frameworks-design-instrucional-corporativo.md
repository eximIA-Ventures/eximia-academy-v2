## 1. OS 8 FRAMEWORKS ANALISADOS

### 1.1 ELC+ 2026 (Enhanced Learning Cycle Plus)

**Origem:** Framework proprietário criado por Hugo Capitelli / eximIA Ventures em janeiro de 2026. Evolução direta do Ciclo de Kolb, expandindo de 4 para 6 estágios.

**Base teórica:** Síntese de 40+ fontes — Kolb (1984), Backward Design, Action Mapping, Cognitive Load Theory (Sweller), Gagné, Kirkpatrick, Bloom, Morris (2020), Heutagogia, Neurociência da aprendizagem.

**Os 6 Estágios:**

| # | Estágio | Verbo | % Tempo | Entregável |
|---|---------|-------|---------|------------|
| 1 | **IMMERSE** | Sentir | 18% | Decisão + justificativa |
| 2 | **REFLECT** | Observar | 12% | 3 aprendizados + 1 dúvida |
| 3 | **CONCEPTUALIZE** | Pensar | 18% | "Como esse conceito muda minha solução?" |
| 4 | **EXPERIMENT** | Fazer | 18% | Versão melhorada + execução |
| 5 | **CALIBRATE** | Validar | 12% | Lista de 3 ajustes |
| 6 | **INTEGRATE** | Internalizar | 22% | Compromisso + plano de aplicação |

**Filosofia:** *"Saídas primeiro, slides depois."* Conteúdo entra como ferramenta de decisão, não como ponto de partida.

**Diferencial:** Único framework que combina base experiencial + validação explícita (CALIBRATE) + transferência explícita (INTEGRATE) + distribuição prescritiva de tempo + entregáveis por etapa + implementação algorítmica.

**Forças:** Resolve as duas maiores falhas de Kolb (falta de validação e transferência); prescritivo com % de tempo; problema-motor como centro; cada etapa com entregável observável; retenção estimada 70-85%.

**Limitações:** Framework novo sem validação acadêmica independente; 6 estágios requerem mais tempo; dependência de problema-motor bem desenhado; exige facilitadores competentes.

**Melhor para:** Treinamentos corporativos de habilidades práticas, mudança comportamental real, cursos de 2+ horas, design instrucional algorítmico.

**Exemplo prático de aula de 60 minutos:**

```
[0-10 min]  SENTIR       — Apresenta caso, aluno decide
[10-17 min] OBSERVAR     — Debrief + 3 aprendizados
[17-28 min] PENSAR       — Mini-aula (1 framework) + comparação
[28-39 min] FAZER        — Refazer decisão com framework
[39-46 min] VALIDAR      — Peer review + rubrica
[46-60 min] INTERNALIZAR — Plano de aplicação + compromisso
```

**Progressão Macro do Curso:**

- Módulo 1: Fundamentos + casos simples (decisões binárias)
- Módulo 2: Variações + ruído (ambiguidade, múltiplas variáveis)
- Módulo 3: Casos com conflito humano (stakeholders com agendas ocultas)
- Módulo 4: Projeto aplicado no mundo real (autonomia total)
- Módulo 5: Demonstração final + reflexão + plano futuro

**4 Dimensões de Progressão:**

- Autonomia do aluno: de guiado para autodirigido
- Realismo do cenário: de simulado para real
- Exigência de qualidade: de "tentativa" para "pronto para produção"
- Pressão de tempo/complexidade: de "sem pressa" para "urgente e ambíguo"

### 1.2 Backward Design (Understanding by Design — UbD)

**Origem:** Grant Wiggins & Jay McTighe, 1998 (ASCD). 2ª edição em 2005.

**Base teórica:** Construtivismo (Dewey, Perkins), Taxonomia de Bloom, Aprendizagem por Transferência.

**Os 3 Estágios:**

| # | Estágio | Pergunta Central |
|---|---------|-----------------|
| 1 | Identificar Resultados Desejados | "O que o aprendiz deve ser capaz de FAZER?" |
| 2 | Determinar Evidências Aceitáveis | "Como saberemos que aprendeu?" |
| 3 | Planejar Experiências de Aprendizagem | "Que atividades levarão ao resultado?" |

**Conceito central:** As 6 Facetas da Compreensão — Explicar, Interpretar, Aplicar, Ter Perspectiva, Empatizar, Autoconhecimento.

**Diferencial:** Inverte a lógica tradicional — começa pelo resultado, não pelo conteúdo. Avaliação é projetada ANTES da instrução.

**Forças:** Foco em resultados mensuráveis; alinhamento com objetivos de negócio; eficiência de tempo (elimina conteúdo desnecessário); transparência para stakeholders.

**Limitações:** Requer mudança de mentalidade; menos prescritivo no "como" desenvolver; originalmente acadêmico (K-12); requer designers experientes.

**Melhor para:** Desenvolvimento de liderança, onboarding, compliance, cursos de alto impacto estratégico onde ROI precisa ser demonstrado.

**Detalhamento dos Estágios:**

**Estágio 1 — Identificar Resultados Desejados:**

- Estabelecem-se objetivos de transferência (o que o aprendiz fará no mundo real)
- Definem-se entendimentos duradouros ("Students will understand that...")
- Formulam-se questões essenciais que guiam à compreensão profunda
- Identificam-se conhecimentos e habilidades específicos necessários

**Estágio 2 — Determinar Evidências Aceitáveis:**

- Avaliação Formativa — mede progresso contínuo durante o curso
- Avaliação Somativa — mede a soma do conhecimento ao final de uma unidade
- Avaliação Baseada em Performance — testa aplicação em situações autênticas

**Estágio 3 — Planejar Experiências de Aprendizagem:**

- Cada atividade deve estar alinhada com os Estágios 1 e 2
- Princípio fundamental: ALINHAMENTO entre os três estágios

**Variações/Adaptações Comuns no Corporativo:**

- UbD + ADDIE Híbrido — backward design dentro da fase de Design do ADDIE
- UbD "Lite" para sprints — versão simplificada: resultado > avaliação > atividade
- Backward Design + Kirkpatrick — combina design reverso com 4 níveis de avaliação

### 1.3 Action Mapping (Cathy Moore)

**Origem:** Cathy Moore, publicado em 2008 no blog, consolidado no livro *"Map It"* (2017).

**Base teórica:** Performance Consulting (Gilbert, Mager), Backward Design, Psicologia Cognitiva.

**As 4 Etapas:**

| # | Etapa | Pergunta |
|---|-------|----------|
| 1 | Meta de Negócio Mensurável | "Que indicador do negócio precisa melhorar?" |
| 2 | Ações Necessárias | "O que as pessoas precisam FAZER? Treinamento é a solução?" |
| 3 | Atividades de Prática | "Como simular decisões reais do trabalho?" |
| 4 | Informação Mínima Necessária | "Qual o mínimo de informação para resolver?" |

**Filosofia:** *"Mude o que as pessoas fazem, não apenas o que elas sabem."*

**Diferencial:** Não é um modelo de desenvolvimento — é um modelo de ANÁLISE e DESIGN que questiona se treinamento é sequer necessário. A Etapa 2 frequentemente revela que treinamento NÃO é a solução.

**Forças:** Foco em resultados do negócio; eliminação de treinamento desnecessário; design enxuto; colaboração efetiva com SMEs; escalável para Fortune 500.

**Limitações:** Inadequado para conteúdo denso/certificações; restrito ao mundo corporativo; não cobre desenvolvimento/implementação; depende de cultura receptiva.

**Melhor para:** Soft skills, compliance comportamental, vendas, atendimento, liderança, segurança no trabalho.

**Detalhamento das Etapas:**

**Etapa 1 — Meta de Negócio Mensurável:**

- Reunião inicial com stakeholders e SMEs
- Pergunta-chave: "Que indicador do negócio precisa melhorar?"
- A meta fica no centro do mapa de ações (visual)
- Exemplo: "Reduzir a perda de clientes em 20% nos próximos 3 meses"

**Etapa 2 — Ações Necessárias:**

- Mapeamento de comportamentos observáveis ("alguém com uma prancheta poderia observar")
- Análise de POR QUE esses comportamentos não estão acontecendo
- Fluxograma de decisão: treinamento é a solução ou é problema de processo/ferramenta/motivação?
- Frequentemente identifica que treinamento NÃO é a solução

**Etapa 3 — Atividades de Prática:**

- Cenários realistas com decisões e consequências
- Formatos: branching scenarios, simulações, role-plays, estudos de caso interativos
- Cada atividade conecta-se diretamente a uma ou mais ações

**Etapa 4 — Informação Mínima Necessária:**

- Somente APÓS desenhar as atividades de prática
- Regra: "Se a informação não suporta diretamente uma atividade, não a inclua!"
- Conteúdo suplementar vai para recursos separados de apoio
- Resultado: cursos dramaticamente mais curtos e focados

### 1.4 SAM (Successive Approximation Model)

**Origem:** Dr. Michael Allen, 2012, livro *"Leaving ADDIE for SAM"* (ATD).

**Base teórica:** Agile Software Development, ISD (Gagné), Construtivismo, modelo CCAF (Context, Challenge, Activity, Feedback).

**As 3 Fases (SAM2):**

| # | Fase | Elemento-Chave |
|---|------|----------------|
| 1 | Preparação | Coleta rápida de informações (breve, não exaustiva) |
| 2 | Design Iterativo | **Savvy Start** (1-2 dias de brainstorming colaborativo) → ciclos de Design/Prototipagem/Revisão |
| 3 | Desenvolvimento Iterativo | Alpha → Beta → Gold (releases incrementais) |

**Filosofia:** *"Progresso antes de perfeição."*

**Diferencial:** O Savvy Start é único — nenhum outro framework tem evento tão estruturado para gerar protótipos funcionais com todos os stakeholders em 1-2 dias.

**Forças:** Velocidade de entrega; flexibilidade; colaboração intensa; redução de risco via protótipos; feedback baseado em evidências; eficiência de custo.

**Limitações:** Risco de soluções instrucionalmente fracas; potencial desperdício em protótipos; prazos imprevisíveis; exige comprometimento contínuo de stakeholders; risco de "iteração infinita".

**Melhor para:** Soft skills, treinamento de software, programas complexos/interativos, organizações com cultura ágil e prazos curtos.

**Detalhamento das Fases:**

**Fase 1 — Preparação:**

- Pesquisas com público-alvo, reuniões com SMEs
- Identificação de recursos disponíveis
- Definição de como o sucesso será medido
- Deve ser BREVE — não é uma análise exaustiva como no ADDIE

**Fase 2 — Design Iterativo:**

- **Savvy Start:** Sessão de 1-2 dias com TODOS os envolvidos (designers, SMEs, gerentes, aprendizes)
- Objetivo: produzir no mínimo 3 protótipos rápidos
- Foco em "possibilidade sobre perfeição" e "amplitude sobre profundidade"
- Ciclo iterativo: Design → Prototipagem → Revisão (repete até consenso)

**Fase 3 — Desenvolvimento Iterativo:**

- Alpha: Design básico com conteúdo rudimentar
- Beta: Versão atualizada incorporando feedback
- Gold: Versão final pronta para implantação

**SAM1 vs SAM2:**

- SAM1: Projetos simples — ciclo iterativo de 3 etapas (Evaluate → Design → Develop), repetido 3 vezes
- SAM2: Projetos complexos — 3 fases completas com Savvy Start

**CCAF (modelo de atividade usado dentro do SAM):**

- Context: Situação realista
- Challenge: Problema a resolver
- Activity: Ação do aprendiz
- Feedback: Consequências das ações

### 1.5 Gagné's 9 Events of Instruction

**Origem:** Robert Gagné, 1965, *"The Conditions of Learning"*. Refinado até 1985.

**Base teórica:** Modelo de processamento de informação cognitiva — cada evento externo corresponde a um processo cognitivo interno. Unificou behaviorismo e cognitivismo.

**Os 9 Eventos:**

| # | Evento | Processo Cognitivo | Exemplo Corporativo |
|---|--------|-------------------|---------------------|
| 1 | Ganhar Atenção | Recepção | Vídeo de ataque ransomware real |
| 2 | Informar Objetivos | Expectativa | "Ao final, você identificará 5 tipos de phishing" |
| 3 | Estimular Recordação | Recuperação | "Que práticas de segurança você já usa?" |
| 4 | Apresentar Conteúdo | Percepção Seletiva | Exemplos visuais de e-mails fraudulentos |
| 5 | Fornecer Orientação | Codificação Semântica | Framework mnemônico "SPAM" |
| 6 | Elicitar Desempenho | Resposta/Prática | Classificar 10 e-mails como legítimo ou phishing |
| 7 | Fornecer Feedback | Reforço | "Acertou 8/10. Nos e-mails 3 e 7, observe..." |
| 8 | Avaliar Desempenho | Recuperação/Avaliação | Teste com cenários novos |
| 9 | Melhorar Retenção e Transferência | Generalização | Guia de referência + simulações mensais |

**Diferencial:** Ancoragem direta nos processos cognitivos internos — a sequência espelha como o cérebro processa informação.

**Forças:** Estrutura clara e replicável; base científica de 60 anos; versatilidade de formato; funciona como checklist de qualidade; avaliação integrada.

**Limitações:** Linear e rígido; intensivo em tempo; favorece aprendizagem passiva; assume homogeneidade; não coloca problemas reais como centro.

**Melhor para:** Treinamentos procedimentais/técnicos, compliance, onboarding, hard skills, conteúdo padronizado em larga escala.

### 1.6 Merrill's First Principles of Instruction

**Origem:** M. David Merrill, 2002, publicado no *Educational Technology Research and Development*.

**Base teórica:** Meta-análise de múltiplas teorias de DI (Gagné, Reigeluth, van Merriënboer, Jonassen). Única meta-teoria do design instrucional.

**Os 5 Princípios:**

| # | Princípio | Essência | Exemplo Corporativo |
|---|-----------|----------|---------------------|
| 0 | **Centrado em Problemas** | Problema real como núcleo organizador | "Sua equipe recebeu um projeto de R$ 2M com 3 stakeholders conflitantes" |
| 1 | Ativação | Ativar conhecimento prévio | "Lembre-se da última vez que um projeto atrasou. O que aconteceu?" |
| 2 | Demonstração | "Show me" em vez de "Tell me" | Vídeo de gerente conduzindo kickoff exemplar |
| 3 | Aplicação | Aprendiz FAZ, não apenas assiste | Planejar kickoff simulado, receber feedback, iterar |
| 4 | Integração | Conhecimento integrado ao mundo real | Liderar próximo kickoff real + retrospectiva com time |

**Diferencial:** Não diz "como" desenhar instrução — diz o que a instrução deve "ter" para ser eficaz. É a síntese dos princípios comuns às teorias que funcionam.

**Forças:** Centrado em problemas reais; alinhado com andragogia; maximiza transferência; flexível e não-prescritivo; baseado em meta-análise; reduz instrução desnecessária.

**Limitações:** Princípios amplos e pouco específicos; exige expertise do designer; intensivo em tempo de desenvolvimento; não é modelo de processo.

**Melhor para:** Habilidades aplicadas, competências complexas, aprendizes adultos com experiência, programas orientados a desempenho, soft skills.

**Progressão de Problemas:** De simples a complexo, cada problema ativando e construindo sobre o anterior.

### 1.7 ADDIE

**Origem:** Florida State University / U.S. Army (1975). Acrônimo "ADDIE" formalizado por Schlegel (1995).

**Base teórica:** Teoria de Sistemas, Behaviorismo, Cognitivismo, Construtivismo.

**As 5 Fases:**

| # | Fase | Foco | Output |
|---|------|------|--------|
| 1 | **Analysis** | Gap de desempenho, público-alvo, restrições | Documento de análise de necessidades |
| 2 | **Design** | Objetivos, avaliação, sequenciamento, blueprint | Storyboard/blueprint instrucional |
| 3 | **Development** | Criação de conteúdo, protótipos, revisão com SMEs | Materiais instrucionais completos |
| 4 | **Implementation** | Entrega, train-the-trainer, logística | Treinamento entregue + feedback inicial |
| 5 | **Evaluation** | Formativa (durante) + Somativa (após) | Relatório de avaliação + plano de melhoria |

**Diferencial:** Framework "pai" do design instrucional moderno — praticamente todos os outros são derivações ou reações ao ADDIE.

**Forças:** Estrutura clara e completa; linguagem universal; análise robusta; flexibilidade de formato; escalabilidade; 50+ anos de validação.

**Limitações:** Rigidez linear (waterfall); consumo de tempo; pouca ênfase em engajamento; fase de Análise frequentemente subestimada por pressão de velocidade.

**Melhor para:** Treinamentos em larga escala, compliance/regulatório, onboarding padronizado, projetos com múltiplos stakeholders, ambientes regulados.

**Variações/Adaptações Comuns no Corporativo:**

- SAM (Successive Approximation Model) — alternativa iterativa
- ADDIE Ágil — ciclos mais curtos com feedback contínuo
- Dick and Carey Model — variação mais detalhada com 10 etapas
- Rapid Prototyping — comprime fases para testar com usuários reais
- ADDIE + Lean — eliminação de desperdícios, MVT (Minimum Viable Training)
- Modelos Híbridos — prática mais comum em 2025-2026

**Exemplos Corporativos:** Toyota, Walmart (VR training), Forças Armadas dos EUA, Marriott (LMS global), Coursera e edX.

### 1.8 Kolb 4-Stage (Experiential Learning Cycle)

**Origem:** David A. Kolb, 1984, *"Experiential Learning"*. Base: Dewey, Lewin, Piaget.

**Os 4 Estágios:**

| # | Estágio | Modo | Exemplo Corporativo |
|---|---------|------|---------------------|
| 1 | Experiência Concreta | Sentir | Role-play com clientes, simulação, shadow |
| 2 | Observação Reflexiva | Observar | Debrief, journaling, discussão em grupo |
| 3 | Conceitualização Abstrata | Pensar | Framework teórico, leitura, mini-aula |
| 4 | Experimentação Ativa | Fazer | Aplicar técnica com clientes reais |

**4 Estilos de Aprendizagem:**

| Estilo | Combinação | Perfil |
|--------|-----------|--------|
| Divergente (CE+RO) | Sentir + Observar | Imaginativo, brainstorming |
| Assimilador (AC+RO) | Pensar + Observar | Lógico, analítico |
| Convergente (AC+AE) | Pensar + Fazer | Resolução técnica |
| Acomodador (CE+AE) | Sentir + Fazer | Hands-on, intuitivo |

> **Nota:** Pesquisas recentes (2024-2025) indicam suporte empírico fraco para estilos de aprendizagem fixos.

**Diferencial:** Framework-pai da aprendizagem experiencial. Referência mais citada (1984), base de praticamente todos os modelos experienciais posteriores.

**Forças:** Acessível e universalmente aplicável; retenção aumentada (~70%); enfatiza aprendizagem ativa; flexível (entra em qualquer estágio).

**Limitações:** Linearidade simplificada; ignora fatores sociais/culturais; suporte empírico fraco para estilos de aprendizagem; sem mecanismo de validação/feedback; sem mecanismo de transferência.

**Melhor para:** Soft skills, workshops presenciais, onboarding, desenvolvimento profissional contínuo.

**Relação com ELC+ 2026:**

```
KOLB ORIGINAL (1984):
  Sentir → Observar → Pensar → Fazer → [recomeça]

ELC+ 2026:
  Sentir → Observar → Pensar → Fazer → VALIDAR → INTERNALIZAR → [recomeça]
                                         (NOVO)     (NOVO)
```

O ELC+ resolve duas lacunas críticas de Kolb:

- **Lacuna 1 — Falta de Validação:** Kolb vai de Experimentação direto para nova Experiência, sem verificar se o aprendizado está correto. CALIBRATE introduz peer review, rubrica e feedback.
- **Lacuna 2 — Falta de Transferência:** Kolb assume que o ciclo simplesmente recomeça. INTEGRATE adiciona teach-back, compromisso de aplicação e plano de ação concreto.

## 2. METODOLOGIAS ADICIONAIS PESQUISADAS

### Frameworks Complementares

#### 2.1 Modelo 70:20:10 (Lombardo, Eichinger & McCall, anos 1980)

**Tipo:** Estratégia de aprendizagem

**Descrição:** 70% experiência prática no trabalho + 20% interação social (mentoria, coaching) + 10% treinamento formal. Os percentuais são diretrizes, não prescrições rígidas.

**Aplicação corporativa:** Ao criar um curso, planejar não apenas o conteúdo formal (10%), mas atividades de aplicação prática (70%) e mecanismos de colaboração social (20%).

**Forças:** Reconhece que a maior parte da aprendizagem acontece fora da sala de aula; alinha-se com "learning in the flow of work".

**Limitações:** Percentuais baseados em pesquisa limitada (200 executivos); difícil de medir; pode subfinanciar treinamento formal.

**Classificação:** Complementar — meta-framework que informa proporção de investimento.

---

#### 2.2 5 Momentos de Necessidade (Mosher & Gottfredson)

**Tipo:** Performance support

**Descrição:** Cinco momentos em que colaboradores precisam de aprendizagem: (1) Aprender pela primeira vez (New), (2) Aprender mais (More), (3) Aplicar no trabalho (Apply), (4) Resolver problemas (Solve), (5) Lidar com mudanças (Change).

**Aplicação corporativa:** Cursos devem incluir não apenas conteúdo de ensino (momentos 1-2), mas job aids, guias de referência rápida e ferramentas de performance support (momentos 3-5).

**Forças:** Foco em performance real; abrange todo o ciclo de necessidade; justifica investimento em EPSS.

**Limitações:** Requer investimento em infraestrutura de suporte; difícil implementar em LMS tradicionais.

**Classificação:** Complementar — combina com modelos de design para criar treinamento + recursos de performance support.

---

#### 2.3 Taxonomia de Bloom Revisada (Anderson & Krathwohl, 2001)

**Tipo:** Objetivos de aprendizagem

**Descrição:** Hierarquia cognitiva com verbos de ação: Lembrar → Compreender → Aplicar → Analisar → Avaliar → Criar. Inclui dimensão de conhecimento: factual, conceitual, procedimental, metacognitivo.

**Aplicação corporativa:** Indispensável para redação de objetivos mensuráveis. Compliance pode focar em "Lembrar/Compreender"; liderança deve atingir "Analisar/Avaliar".

**Forças:** Linguagem universal; progressão clara do simples ao complexo; facilita alinhamento objetivo-atividade-avaliação.

**Limitações:** Pode ser usada mecanicamente; hierarquia linear nem sempre reflete aprendizagem real; focada no domínio cognitivo.

**Classificação:** Complementar — ferramenta de classificação que se integra a qualquer modelo.

---

#### 2.4 TPACK (Mishra & Koehler, 2006)

**Tipo:** Competência de educadores

**Descrição:** Interseção de Conhecimento Tecnológico (TK) + Pedagógico (PK) + de Conteúdo (CK). A combinação dos três forma o TPACK.

**Aplicação corporativa:** Garante que escolha de tecnologia (LMS, IA, VR) seja informada pela pedagogia e conteúdo. Evita erro de escolher tecnologia sem considerar objetivo pedagógico.

**Forças:** Impede decisões tecnológicas desconectadas; útil para diagnosticar gaps em equipes de L&D.

**Limitações:** Originalmente para educação formal; abstrato para aplicação prática direta.

**Classificação:** Complementar — framework de competência, não de design.

---

#### 2.5 Kirkpatrick 4 Níveis (1959, atualizado 2016)

**Tipo:** Avaliação

**Descrição:** Quatro níveis de avaliação: Nível 1 (Reação — satisfação), Nível 2 (Aprendizado — conhecimento adquirido), Nível 3 (Comportamento — transferência para trabalho), Nível 4 (Resultados — impacto no negócio).

**Aplicação corporativa:** Fundamental para demonstrar ROI. Planejar avaliações desde o início: pesquisas de satisfação (N1), pré/pós-testes (N2), observação de desempenho (N3), métricas de negócio (N4).

**Forças:** Linguagem universal; facilita comunicação com stakeholders; "New World" adiciona planejamento reverso.

**Limitações:** Níveis 3 e 4 são complexos e caros de medir; correlação entre níveis não é linear.

**Classificação:** Complementar — exclusivamente modelo de avaliação.

---

#### 2.6 Andragogia (Malcolm Knowles, 1968)

**Tipo:** Teoria de adultos

**Descrição:** Seis princípios da aprendizagem adulta: (1) Necessidade de saber (por que aprender?), (2) Autoconceito (autonomia), (3) Experiência como recurso, (4) Prontidão para aprender (relevância), (5) Orientação para problemas, (6) Motivação intrínseca.

**Aplicação corporativa:** Absolutamente central. Os princípios devem permear todo o design: explicar o "por que", oferecer escolhas, conectar a experiências, garantir aplicabilidade, usar cenários reais, apelar para motivações intrínsecas.

**Forças:** Especificamente focada em adultos (público corporativo); princípios pragmáticos e aplicáveis.

**Limitações:** Pode superestimar autodirecionamento; princípios genéricos; ignora diferenças culturais.

**Classificação:** Complementar — deve informar qualquer modelo de design corporativo.

---

#### 2.7 Teoria da Aprendizagem Social (Bandura, 1977)

**Tipo:** Teoria de aprendizagem

**Descrição:** Aprendizagem por observação: Atenção → Retenção → Reprodução → Motivação. Inclui conceito de autoeficácia.

**Aplicação corporativa:** Fundamenta programas de mentoria, job shadowing, comunidades de prática. Inclui vídeos de experts, estudos de caso, fóruns de discussão.

**Forças:** Base sólida para aprendizagem social/colaborativa; conceito de autoeficácia é relevante para engajamento.

**Limitações:** Difícil controlar qualidade; modelos negativos também são aprendidos; resultados difíceis de medir.

**Classificação:** Complementar — informa estratégias sociais dentro de outros modelos.

---

#### 2.8 Construtivismo / ZDP (Vygotsky)

**Tipo:** Teoria de aprendizagem

**Descrição:** Zona de Desenvolvimento Proximal = distância entre o que o aprendiz faz sozinho e o que faz com orientação. Scaffolding = suporte temporário gradualmente removido.

**Aplicação corporativa:** Fundamenta trilhas adaptativas, programas de mentoria decrescente, sistemas de IA que calibram dificuldade e suporte em tempo real.

**Forças:** Base para aprendizagem adaptativa e personalizada; scaffolding é intuitivo e aplicável.

**Limitações:** Identificar ZDP individual é complexo em escala; difícil operacionalizar em treinamentos auto-dirigidos.

**Classificação:** Complementar — informa estratégias de scaffolding e adaptatividade.

---

#### 2.9 Microlearning

**Tipo:** Formato/entrega

**Descrição:** Unidades de 3-10 minutos focadas em um único objetivo. Formatos: micro-vídeos, flashcards, quizzes, micro-simulações. Mercado: USD 2.96B em 2025, CAGR 13.5%.

**Aplicação corporativa:** 93% das organizações acreditam que será parte essencial de seus programas. Taxas de conclusão ~80%, retenção 25-60% superior. Em 2025-2026: Microlearning 3.0 com IA adaptativa.

**Forças:** Altas taxas de conclusão e retenção; compatível com mobile; desenvolvimento rápido e barato.

**Limitações:** Inadequado para competências complexas; risco de fragmentação; pode gerar "fadiga de conteúdo".

**Classificação:** Complementar — estratégia de formato que se integra a qualquer modelo.

---

#### 2.10 Design Thinking para Aprendizagem

**Tipo:** Discovery/ideação

**Descrição:** Aplicação de Design Thinking ao design de experiências de aprendizagem: Empatizar → Definir → Idear → Prototipar → Testar.

**Aplicação corporativa:** Transforma "o que a liderança pediu" para "o que os colaboradores realmente precisam". Inclui entrevistas, observação, personas, prototipagem.

**Forças:** Coloca aprendiz no centro; revela necessidades não articuladas; reduz desperdício.

**Limitações:** Pode ser percebido como "lento"; falta rigor pedagógico intrínseco; exige facilitadores experientes.

**Classificação:** Complementar — abordagem de discovery que se integra à fase de análise.

---

#### 2.11 xAPI (Experience API)

**Tipo:** Infraestrutura técnica

**Descrição:** Especificação que rastreia experiências de aprendizagem em qualquer contexto ("Ator + Verbo + Objeto"). Dados armazenados em Learning Record Store (LRS).

**Aplicação corporativa:** Mede aprendizagem além de "conclusão de curso". Rastreia performance em simulações, comunidades, aplicação real. Transição SCORM → xAPI acelerando.

**Forças:** Libera medição de contextos que SCORM não alcança; integração com BI e analytics preditivo.

**Limitações:** Requer investimento técnico (LRS, integrações); transição ainda em andamento.

**Classificação:** Complementar — infraestrutura técnica de medição.

---

### Frameworks Autônomos Adicionais

#### 2.12 Dick & Carey (1978)

**Tipo:** Design instrucional sistemático

**Descrição:** 10 etapas rigorosas: (1) identificar metas, (2) análise instrucional, (3) analisar aprendizes, (4) objetivos de desempenho, (5) instrumentos de avaliação, (6) estratégia instrucional, (7) materiais, (8) avaliação formativa, (9) revisar instrução, (10) avaliação somativa.

**Aplicação corporativa:** Ideal para ambientes regulados (saúde, aviação, finanças) onde cada decisão deve ser documentada e validada.

**Forças:** Extremamente rigoroso; visão sistêmica; ideal para auditabilidade.

**Limitações:** Processo longo e pesado; excessivo para treinamentos simples; linearidade dificulta adaptação.

**Classificação:** Autônomo — modelo completo, comparável ao ADDIE com maior detalhamento.

---

#### 2.13 4C/ID (van Merriënboer, 1997)

**Tipo:** Competências complexas

**Descrição:** Quatro componentes: (a) Learning Tasks — tarefas completas e autênticas em complexidade crescente, (b) Supportive Information — modelos mentais e estratégias, (c) JIT Information — instruções procedimentais no momento exato, (d) Part-Task Practice — exercícios para automatizar rotinas.

**Aplicação corporativa:** Ideal para diagnóstico médico, resolução de problemas técnicos, liderança, vendas consultivas, gestão de projetos.

**Forças:** Mais rigoroso para competências complexas; whole-task approach; forte base em ciência cognitiva.

**Limitações:** Trabalhoso; não adequado para conteúdo simples; requer task analysis aprofundada.

**Classificação:** Autônomo — modelo completo com 10 passos (Ten Steps to Complex Learning).

---

#### 2.14 Agile Learning Design

**Tipo:** Gestão de projeto

**Descrição:** Sprints de 2 semanas, entregas incrementais, feedback contínuo. Inclui LLAMA (Lean Learning Agile Methods and Approaches).

**Aplicação corporativa:** Equipes de L&D entregam módulos incrementalmente, coletando feedback a cada iteração.

**Forças:** Velocidade; adaptabilidade; colaboração multidisciplinar; reduz risco.

**Limitações:** Pode sacrificar rigor pedagógico; requer mudança cultural; stakeholders precisam estar disponíveis.

**Classificação:** Autônomo como gestão de projeto — substitui gestão linear do ADDIE, mas precisa de frameworks pedagógicos.

---

#### 2.15 LXD — Learning Experience Design (Niels Floor, 2007)

**Tipo:** Filosofia de design

**Descrição:** UX design + game design + psicologia cognitiva aplicados a aprendizagem. 9 regras: aprendemos por experiência; cada experiência é única; aprendizagem é contínua/dinâmica/holística; emoção influencia; centralismo humano; interatividade é eficaz; experiências devem ser positivas, pessoais e profundas.

**Aplicação corporativa:** Substitui mentalidade de "entrega de conteúdo" por "design de experiência". Usa Learning Experience Canvas, personas, mapas de empatia.

**Forças:** Rigor de UX para design educacional; convergência com IA para experiências adaptativas.

**Limitações:** Pode ser abstrato para designers tradicionais; carece de prescrições pedagógicas específicas.

**Classificação:** Autônomo como filosofia — funciona melhor combinado com modelos estruturados.

### 3.2 Matriz de Adequação por Tipo de Treinamento Corporativo

| Tipo de Treinamento | Framework Primário | Complementar |
|---------------------|-------------------|--------------|
| **Compliance/regulatório** | Gagné (estrutura obrigatória) | ADDIE (gestão) + Kirkpatrick (avaliação) |
| **Onboarding técnico** | Gagné (passo-a-passo) | Merrill (cenários reais) + Bloom (objetivos) |
| **Desenvolvimento de liderança** | **ELC+** ou **Merrill** (problemas reais) | Backward Design (resultados) + Kirkpatrick (L3/L4) |
| **Vendas e negociação** | **ELC+** ou **Merrill** (cenários de cliente) | Action Mapping (comportamento) + Gagné (módulos) |
| **Soft skills** | **ELC+** (ciclo completo) | Merrill (princípios) + Kolb (experiencial) |
| **Treinamento de software** | **SAM** (iterativo, muda frequentemente) | Gagné (procedimentos) + Microlearning (just-in-time) |
| **Programas estratégicos (alto ROI)** | **Backward Design** + **ELC+** | Kirkpatrick (L4) + Action Mapping (foco) |
| **Treinamento técnico especializado** | **Gagné** + **4C/ID** | Bloom (progressão) + xAPI (tracking) |
| **Redesenho de treinamento ineficaz** | **Action Mapping** (questiona a premissa) | Design Thinking (empatia) + SAM (prototipagem) |
| **Programas em larga escala** | **ADDIE** (padronização) | Gagné (dentro de cada módulo) + Kirkpatrick |

### 3.5 Quando Combinar Frameworks

A tendência dominante em 2025-2026 é a **hibridização**. Equipes maduras não adotam um framework de forma pura:

**Combinação recomendada para máxima eficácia:**

```
┌─────────────────────────────────────────────────────────┐
│  CAMADA 1 — ESTRATÉGIA                                  │
│  70:20:10 + 5 Momentos de Necessidade + Andragogia      │
│  (Define o ecossistema de aprendizagem)                  │
├─────────────────────────────────────────────────────────┤
│  CAMADA 2 — DESIGN DO PROGRAMA                          │
│  Backward Design (resultados) + Action Mapping (foco)   │
│  (Define O QUE ensinar e SE treinamento é a solução)    │
├─────────────────────────────────────────────────────────┤
│  CAMADA 3 — DESIGN DO MÓDULO                            │
│  ELC+ 2026 (ciclo experiencial) OU Merrill (princípios) │
│  + Gagné (checklist de completude)                       │
│  (Define COMO estruturar cada experiência)               │
├─────────────────────────────────────────────────────────┤
│  CAMADA 4 — PROCESSO DE DESENVOLVIMENTO                 │
│  SAM (iterativo) OU ADDIE (estruturado)                 │
│  + Agile Learning Design (sprints)                       │
│  (Define COMO produzir o treinamento)                    │
├─────────────────────────────────────────────────────────┤
│  CAMADA 5 — AVALIAÇÃO                                   │
│  Kirkpatrick 4 Níveis + Bloom (objetivos) + xAPI        │
│  (Define COMO medir o impacto)                           │
└─────────────────────────────────────────────────────────┘
```

| **Avaliação de impacto** | **Kirkpatrick** — mede reação, aprendizado, comportamento e resultados |

| **Formato de entrega** | **Microlearning + 5MoN** — suporte no fluxo de trabalho |
